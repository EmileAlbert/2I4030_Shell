                                                                                      int argSize = sizeof(argmnt);

                                                                                      sizeof() is not a function that returns the number of elements in an array, it is a language builtin which evaluates (at compile-time) to the byte size of the type or variable given. Since argmnt is a char **, that is 8 bytes (on most 64 bit systems). argmnt is probably (if this is my shell) a NULL-terminated array, so you need to write a function to count the number of items in the array.

                                                                                                                                                          • Stephen Brennan Author rawi sader • a year ago

                                                                                                                                                            Sounds like you're creating a special built-in command that simply calls execlp() with the Java command and class name. All command line args need to be explicitly passed into exec(). So, for instance, if you ran the command "ls -l", lsh would break it into the arg list ["ls", "-l"], and then it would call execvp() with that list. This means that the command line arguments from the shell make it to normal programs.

                                                                                                                                                            Let's take a look at what happens with a built-in. Say you ran "mybuiltin arg1 arg2". Again, lsh would split this into the argument list ["mybuiltin", "arg1", "arg2"]. Assuming you have implemented a builtin named "mybuiltin", the builtin function would be called with that list. However, your builtin function then just calls execlp("java", "java", "nameofclass", NULL), without including that list of arguments! So of course your Java program never gets its arguments, and thus it fails with an IndexOutOfBoundsException.

                                                                                                                                                            What you'll need to do is make sure you call exec (some version of it) so that your first arguments are "java" "nameofclass", but then the remaining arguments are taken from the args list your builtin function is called with. There are a few approaches you could take with this, but the most straightforward one is probably to create an array "newargs", which has room for "java", "nameofclass", and then all of the arguments that come after the name of your builtin in the args array (plus a NULL terminator). Copy all the arguments into this array, make sure it's NULL terminated, and then use execvp("java", newargs), and you should be golden!

                                                                                                                                                            Hope that all makes sense!
